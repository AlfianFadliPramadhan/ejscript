Ejscript Change Log

This change log chronicles the features and fixes in each Ejscript release.

While Ejscript has been used in many commercial products, we want to effectively communicate our commitment to versioning and stability of APIs. We are adopting the Semantic Versioning Scheme and have reset the Ejscript version number back to 0.8.0. This is because the public API for Ejscript is still evolving and we are not yet ready to make firm commitments to backwards compatibility. When the public API is stable, we will bump the version number to 1.0.0.

Changes in 0.8.0

Major New Features

  • Packages. Packages provide a means and format for distributing Ejscript programs and libraries. An Ejscript package is a cohesive wrapping of a collection of modules, code and other assets into a single distributable unit. It provides the basis for convenient delivery, installation and management of CommonJS components.
  • Decouple core components into packages. Some of the packages provided include: database, sqlite, database ORM mapper, Appweb bindings, web page templating, MVC web framework and XML support.

  • Package manager command jem.
  • Run pure scripts as web commands supporting the JSGI (Rack/Jack style) interface.
  • Stand-alone web templating command.
  • Enhanced caching for scripts and web pages. Cached, pre-compiled code stored in common cache directory.
  • Events, Timers and Async execution.
  • Sockets class.
  • Optimized VM performance by ~20%, web page serving performance by 130% and event handling by 270%.
  • Support for the CommonJS Secure Modules interface specification.
  • Support for the CommonJS JSGI web interface specification.
  • Support for the CommonJS Package specification.

Minor Features

  • Inline compilation of web pages and web scripts.
  • Support for an "ejsrc" configuration file to tailor the execution environment.
  • MD5 checksum function.

Fixes

  • Update to use latest Appweb release.

API Changes

  • ejsweb command renamed to ejsmvc.
  • ejswebserver command renamed to ejsweb.

    • ejs Package

    • EjsVar renamed to EO
    • ejsXXXVar renamed to ejsXXX or ejsXXXObj where appropriate.

    ejs.xml Package

    • Need require ejs.xml to use XML

    ejs.web.appweb Package

    • Not mapping "-" to "_" in headers anymore, but now mapped to lower case.

    ejs.web.mvc Package

    • redirect renamed to redirectAction
    • redirectUrl renamed to redirect
