QualifiersPropertyTypeDescription
public const get absolutePathAn equivalent absolute path equivalent for the current path. The path is normalized.
public const get accessedDateWhen the file represented by the path was last accessed. Set to null if the file does not exist.
public const get basenamePathThe base of portion of the path. The base portion is the trailing portion without any directory elements.
public const get componentsArrayPath components. This is the path converted to an absolute path and then broken into components for each directory level. It is set to an array object with an element for each segment of the path. The first element represents the file system root and will be the empty string or drive spec on a windows like systems. Array.join("/") can be used to put components back into a complete path.
public const get createdDateWhen the file represented by the path was created. Set to null if the file does not exist.
public const get dirnamePathThe directory portion of a file. The directory portion is the leading portion including all directory elements and excluding the base name. On some systems, it will include a drive specifier. See also parent which will determine the parent directory of relative paths.
public const get existsBooleanDoes a file exist for this path.
public const get extensionStringFile extension portion of the path. The file extension is the portion after (and not including) the last ".".
public const get hasDriveBooleanDoes the file path have a drive spec (C:) in it's name. Only relevant on Windows like systems.
public const get isAbsoluteBooleanIs the path absolute, i.e. Beginning at the file system's root.
public const get isDirBooleanIs the path a directory.
public const get isLinkBooleanIs the path is a symbolic link file. Not available on some platforms such as Windows and VxWorks.
public const get isRegularBooleanIs the path a regular file.
public const get isRelativeBooleanIs the path is relative and not absolute.
public const get lengthNumberThe length of the path name in bytes. Note: this is not the file size. For that, use Path.size.
public const get linkTargetPathThe target pointed to if this path is a symbolic link. Not available on some platforms such as Windows and VxWorks. If the path is not a symbolic link, it is set to null.
public const get mimeTypeStringGet the mime type for a path or extension string. The path's extension is used to lookup the corresponding mime type.
public const get modifiedDateWhen the file represented by the path was last modified. Set to null if the file does not exist.
public const get nameStringName of the Path as a string. This is the same as toString().
public const get naturalPathNatural (native) respresentation of the path. This uses the default O/S file system path separator, this is "\" on windows and "/" on unix and is normalized.
public const get normalizePathNormalized representation of the path. This removes "segment/.." and "./" components. Separators are made uniform by converting all separators to be the same as the first separator found in the path. Note: the result is not converted to an absolute path.
public const get parentPathThe parent directory of path. This may be an absolute path if there are no parent directories in the path.
public const get portablePathThe path in a portable (like Unix) representation. This uses "/" separators. The value is is normalized and the separators are mapped to "/".
public const get relativePathThat path in a form relative to the application's current working directory. The value is normalized.
public const get separatorStringThe path separator for this path. This will return the first valid path separator used by the path or the default file system path separator if the path has no separators. On Windows, a path may contain "/" and "\" separators. This will be set to a string containing the first separator found in the path. Will typically be either "/" or "/\\" depending on the path, platform and file system. Use natural or portable to create a new path with different path separators.
public const get sizeNumberSize of the file associated with this Path object. Set to the number of bytes in the file or -1 if the size determination failed.
