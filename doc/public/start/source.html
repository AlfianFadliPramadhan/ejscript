Building Ejscript from Source

Ejscript releases are available in binary and source code distributions. It is your choice which distribution you choose, but the source code distribution has several advantages:

  • You can recompile Ejscript to optimize it for your specific system environment.
  • You can customize Ejscript via configuration options to select the specific feature set you require.
  • You can minimize the memory footprint and decrease the securable surface area.

Supported Platforms

Several build environments are supported:

  • Linux — Linux 2.6 with GNU C/C++
  • Windows — Microsoft Windows 7 with Visual Studio 2010 or later
  • Mac OS X — Mac OS X 10.8 (Mountain Lion) or later

The source code has been designed to run on FreeBSD, Linux, Mac OS X, VxWorks and other operating systems. It has been tested under Fedora and Ubuntu Linux based on the Linux 2.6 Kernel, Microsoft Windows 7, 8 (32 and 64-bit), VxWorks 6.X and MAC OS X 10.8.

Accessing the Source Code

With each Ejscript release, a complete source code snapshot is provided. You can download a source package from the Ejscript web site or you can retrieve it from the Ejscript GitHub repository. If you download from the repository, please use the master branch for the most recent stable snapshot. For less stable daily builds, use the dev branch.

Installing Packages

Ejscript can be extended via third-party packages that are published in the online Pak Catalog. There are many packages including:

  • OpenSSL
  • PHP
  • Angular
  • ESP for Angular
  • ESP Angular MVC

These can be installed via simple Pak commands. For example:

pak install esp-angular-mvc

See the Pak Web Site to read about Pak and to Download Pak.

Building Ejscript from Source

The Ejscript source distribution contains all the required source files, headers, and test framework to reconfigure, build, and verify Ejscript. The software supports three methods for building from source.

  • Make
  • MakeMe
  • Visual Studio or Xcode

Building via make is the simplest way to build. Use make if you want a default build or if you need to integrate Ejscript with an existing make-based build system. Build with MakeMe if you want to customize or configure the software for your system, or if you need to cross-compile for another operating system or architecture. Use the Visual Studio or Xcode if you are most comfortable with these tools and want a default build.
